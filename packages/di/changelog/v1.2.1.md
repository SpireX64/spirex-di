# SpireX/DI — v1.2.1 Change Log

## Fixed: Disposing the root container scope

The root scope has **no parent**. Previous releases still attempted to remove the scope from the parent’s child map during `dispose()`, which caused a runtime error when disposing the built container (the object returned from `build()`):

```ts
const container = diBuilder().bindInstance("x", 1).build();

container.dispose();
// TypeError: Cannot read properties of undefined (reading '…')
```

Disposal now updates the parent registry **only when a parent scope exists**, so `dispose()` on the root completes successfully and local instances are still cleaned up as before.

```ts
const container = diBuilder().bindInstance("x", 1).build();

container.dispose();
expect(container.isDisposed).toBe(true);
```

## Fixed: Removing a disposed child from the parent scope map

When a child scope is disposed, it must unregister from its parent so that `hasChildScope(id)` becomes `false` and a **new** child with the same `id` can be opened later.

The unregister step now deletes the entry **only if** the map still points at **this** scope instance (`parent[$scopes].get(this.id) === this`). That avoids removing another scope object that may have been registered under the same id after the previous one was disposed, and keeps the parent’s child map consistent.

```ts
const session = root.scope("session");
session.dispose();

const nextSession = root.scope("session");
// nextSession is a new scope; root.hasChildScope("session") is true again
```

------------------
## Summary

v1.2.1 is a **patch** release focused on **scope disposal correctness**: the root container can be disposed without throwing, child scopes are detached from their parent reliably, and reopening a child scope with the same identifier behaves as expected. No breaking changes; public APIs are unchanged.
